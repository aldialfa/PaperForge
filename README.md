## 📖 What is PaperForge?

PaperForge is an easy-to-use tool for writing academic papers. It helps with every step from coming up with ideas to creating a final PDF with LaTeX. You don’t need to be a programmer or know LaTeX. The system guides you through idea generation, literature search, experiment execution, adding results, and compiling the paper. It works with multiple language models, supports remote usage via SSH, and keeps your work up to date automatically.

## 🖥️ System Requirements

Before you start, check these requirements to make sure PaperForge will run smoothly on your Windows PC.

- Windows 10 or later (64-bit)
- At least 8 GB of RAM
- 4 GB free disk space
- Internet connection for AI features and remote execution
- Optional: SSH access if you want to use remote training or experiment execution

If you are unsure about any of these, your PC is likely compatible if it can run recent software like web browsers or office apps.

## ⚙️ Setting Up PaperForge for Your First Paper

After opening PaperForge, you will see a simple dashboard. This is your main workspace. To start:

- Click **New Project**.
- Enter a title for your paper.
- Choose the AI model you want to use for idea generation.
- If you want, connect to a remote computer using SSH under **Settings**. This lets you run heavy tasks elsewhere.

PaperForge will then help you with the following steps in order:

- Idea brainstorming
- Finding and summarizing papers
- Planning your experiments
- Running or importing experimental results
- Writing and compiling the final paper in LaTeX format

You can pause and return to any step later. PaperForge saves your progress automatically.

## 📂 How PaperForge Organizes Your Work

Each project is saved in its own folder. This folder contains:

- Notes and outlines
- Retrieved papers and summaries
- Experiment scripts and results
- The LaTeX source files
- The final PDF paper

You can find your project folders under the default directory shown in **Settings** or choose a custom location.

## 🔐 Remote Execution with SSH

If you have a remote server or computer, you can connect through SSH:

- Go to **Settings** > **Remote Execution**
- Enter your server address, username, and password or private key
- Test the connection to ensure it works
- Once connected, PaperForge can run experiments and train models remotely

This frees your local PC from heavy processing tasks.

## 🛠 Troubleshooting Common Issues

If PaperForge does not start or crashes:

- Restart your PC and try again
- Make sure you have the latest Windows updates installed
- Check your antivirus is not blocking PaperForge
- Reinstall the app if needed by downloading again from the GitHub page

If AI features lag or fail, check your internet connection.

For SSH issues:

- Confirm your server is online and accessible
- Double-check the SSH credentials
- Use a standard SSH client like PuTTY to test access manually
